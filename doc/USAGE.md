# 台股 Treemap 使用說明

本文件說明如何安裝、設定與操作「台股即時 Treemap」監控系統。

---

## 1. 這個程式做什麼？

本系統串接**永豐金 Shioaji API**，讀取 `stock_index/` 內的全市場標的清單（上市、上櫃、ETF），取得即時行情後，以 **四個 Treemap 熱力圖**依序呈現：

1. **上市 + 上櫃股票**
2. **上市股票**
3. **上櫃股票**
4. **ETF**

每個 Treemap 的視覺編碼一致：

- **方格大小**：成交額或成交量（可切換，四圖同步）
- **方格顏色**：漲跌幅（紅漲、綠跌）
- **方格標籤**：股票名稱、現價、漲跌幅（三行）

適合用來快速掌握台股整體與分市場狀況，也可將行情匯出成 `.txt` 檔案供後續分析。

## 3. 安裝與設定

### 3.1 啟用 Conda 環境

若尚未建立 `sinotrade` 環境：

```powershell
conda env update -n sinotrade -f environment.yml --prune
conda activate sinotrade
```

### 3.2 設定 API 金鑰

1. 複製範本檔：

```powershell
copy .env.example .env
```

2. 編輯 `.env`，填入永豐金 Shioaji API 金鑰：

```
SJ_API_KEY=你的_API_KEY
SJ_SEC_KEY=你的_SECRET_KEY
SJ_PRODUCTION=true
```

| 變數 | 說明 |
|------|------|
| `SJ_API_KEY` | API 金鑰 |
| `SJ_SEC_KEY` | Secret 金鑰 |
| `SJ_PRODUCTION` | `true` = 正式環境；`false` = 模擬模式 |

> **注意**：`.env` 含敏感資訊，請勿上傳至 GitHub 或公開分享。

### 3.3 確認標的清單

程式會讀取 `stock_index/` 內三份清單：

- `Listed_Company_list.txt` — 上市
- `OTC_Company_list.txt` — 上櫃
- `ETF_list.txt` — ETF

若需更新標的，請替換對應清單檔（tab 分隔格式）。

## 4. 啟動程式

在專案根目錄執行：

```powershell
conda activate sinotrade
cd c:\Users\TT\Documents\cursor\stock_treemap
$env:PYTHONIOENCODING="utf-8"
streamlit run main.py
```

瀏覽器會自動開啟（預設 `http://localhost:8501`）。

**首次啟動**時，系統會登入 Shioaji 並載入商品檔，約需 **10～15 秒**，請耐心等候。

## 5. 介面操作

### 5.1 主畫面

| 區塊 | 說明 |
|------|------|
| 標題列 | 顯示「台股即時 Treemap」 |
| 全域摘要 | 有成交標的數、平均漲跌幅、上漲/下跌家數（含側欄勾選的所有市場） |
| 四個 Treemap | 依序顯示上市+上櫃、上市、上櫃、ETF；各面板上方有該分類的摘要統計 |
| 行情表格 | 展開後可查看成交額前 100 名 |

更新行情後，頁面會向下捲動瀏覽四張圖。每張圖可獨立縮放、點擊鑽入產業或個股層級。

### 5.2 側欄控制面板

#### 市場篩選

勾選要納入的市場：

- **上市** — 含一般上市與創新板
- **上櫃**
- **ETF**

取消勾選可縮小抓取範圍，減少 API 流量消耗。

#### Treemap 大小

- **成交額**（預設）：方格越大代表成交金額越高
- **成交量**：方格越大代表成交張數越多

此設定會同步套用至四個 Treemap 面板。

#### 漲跌幅顏色範圍 (±%)

調整顏色飽和區間，預設 ±5%。超出此範圍的漲跌幅會顯示為最深紅或最深綠。

#### 更新行情

按 **「更新行情」** 按鈕，向 Shioaji 抓取目前勾選市場的全部標的快照。

- 全市場約 2,185 檔，分 5 批抓取，約需 **數十秒**
- 完成後主畫面會顯示最後更新時間
- 若部分代號找不到商品檔，會以黃色警告列出（通常為 5 檔左右）

#### 自動刷新

- 預設**關閉**，建議手動刷新以節省 API 流量
- 開啟後，每隔設定秒數自動重新抓取（最小 **15 秒**）
- 當 API 剩餘流量低於 50MB 時，系統會警告並建議停止自動刷新

#### 匯出 .txt

抓取行情後，按 **「匯出 .txt（上市/上櫃/ETF）」**，會依分類輸出至 `stock_data/`：

```
stock_data/
├── listed_20260615_143000.txt   # 上市
├── otc_20260615_143000.txt      # 上櫃
└── etf_20260615_143000.txt      # ETF
```

空類別不產檔。檔案為 UTF-8 編碼，含代號、名稱、收盤價、漲跌幅、成交額等欄位。

#### API 流量

顯示目前已用 / 上限 / 剩餘流量（MB）及連線數。新帳戶每日上限約 **500MB**，每日 **08:00** 重置。

## 6. Treemap 解讀方式

### 四個面板與分層結構

主畫面不是單一「全市場」Treemap，而是 **四張獨立圖表**，各自聚焦不同標的群：

| 面板 | 內容 | 分層結構 |
|------|------|----------|
| 1. 上市 + 上櫃 | 不含 ETF | 根 → 市場 → 產業 → 個股 |
| 2. 上市 | 僅上市（含創新板） | 根 → 市場 → 產業 → 個股 |
| 3. 上櫃 | 僅上櫃 | 根 → 產業 → 個股 |
| 4. ETF | 僅 ETF | 根 → ETF 方格 |

範例（面板 1）：

```
上市+上櫃
 ├── 上市
 │    ├── 半導體業
 │    │    └── 台積電、聯發科 …
 │    └── 金融保險業
 │         └── …
 └── 上櫃
      └── …
```

點擊方格可鑽入下一層；雙擊空白處可返回上層。若側欄未勾選某市場（例如 ETF），該類別不會被抓取，對應面板會顯示「此分類尚無成交資料。」

### 顏色（漲跌幅）

| 顏色 | 意義 |
|------|------|
| 紅色 | 上漲 |
| 灰色 | 平盤附近 |
| 綠色 | 下跌 |

### 方格標籤（三行）

1. **股票名稱**
2. **現價**
3. **漲跌幅**（如 `+2.16%`）

方格太小時文字會自動隱藏，避免重疊。

### 滑鼠懸停（Hover）

可查看代號、現價、漲跌幅、成交額、成交量等詳細資訊。

## 7. API 限流注意事項

本系統已內建防限流機制，但請留意以下官方限制：

| 限制 | 說明 |
|------|------|
| 每日流量 | 新帳戶 500MB/日，08:00 重置 |
| 查詢頻率 | snapshots 等行情查詢，5 秒內最多 50 次 |
| 連線數 | 同一 ID 最多 5 個連線 |
| 登入次數 | 每日最多 1000 次 |

**建議用法：**

- 盤中才開啟自動刷新，非交易時段避免空跑
- 優先使用手動「更新行情」
- 留意側欄流量進度，接近上限時停止刷新
- 不要同時開多個 Streamlit 視窗（會佔用連線數）

若超過限制，行情可能回傳空值，嚴重時 API 會暫停服務 1 分鐘。

## 9. 常見問題

### Q：啟動後顯示「缺少 SJ_API_KEY 或 SJ_SEC_KEY」

`.env` 未設定或路徑錯誤。確認 `.env` 在專案根目錄，且金鑰已正確填入。

### Q：抓取失敗：`'NoneType' object has no attribute 'dict'`

少數代號在 Shioaji 商品檔中不存在（如 1435、1589 等），系統會自動略過並列入警告，不影響其餘標的。

### Q：Treemap 顯示「尚無行情資料」

尚未按「更新行情」，或目前非交易時段、標的無成交。請先按側欄按鈕抓取。

### Q：某個面板顯示「此分類尚無成交資料」

代表該分類在已抓取資料中沒有 `total_amount > 0` 的標的。常見原因：側欄未勾選該市場、盤中該類別無成交、或 ETF 在非交易時段無報價。

### Q：首次登入很慢

正常現象。Shioaji 需下載全市場商品檔（約上萬筆），首次約 10～15 秒，之後同一 session 會快取。

### Q：中文顯示亂碼

PowerShell 請設定 `$env:PYTHONIOENCODING="utf-8"` 後再執行。避免使用 `conda run`（在 Windows cp950 環境下可能出現編碼問題）。

### Q：匯出的 .txt 在哪裡？

`stock_data/` 資料夾，檔名格式 `{listed|otc|etf}_{YYYYMMDD_HHMMSS}.txt`。
